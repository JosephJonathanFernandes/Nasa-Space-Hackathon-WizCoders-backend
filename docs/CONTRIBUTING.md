# Contributing Guidelines

Thank you for considering contributing!

## How to Contribute
- Fork the repository and create a new branch
- Make your changes with clear, descriptive commits
- Ensure code passes linting and tests
- Submit a pull request with a clear description

## Code Style
- Follow PEP8 and Clean Code principles
- Use meaningful names and docstrings
- Avoid hardcoded secrets; use environment variables

## Security
- Never commit credentials or secrets
- Validate and sanitize all inputs

## Tests
- Add or update unit/integration tests for your changes
- Run all tests before submitting

## Communication
- Be respectful and collaborative
- See CODE_OF_CONDUCT.md for behavior standards
